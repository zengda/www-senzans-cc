**⚠️ 核心约束：**
- **生产环境运行于**：Cloudflare Workers + Pages

## 三、实施方案

### 3.1 本地开发流程

```bash
# 安装依赖
npm install

# 启动开发服务器 (http://localhost:4321)
npm run dev

# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 部署到 Cloudflare
npm run deploy
```

### 3.2 项目结构

```
www_senzans_cc/
├── public/              # 静态资源 (图片、字体等)
│   ├── fonts/
│   └── blog-placeholder-*.jpg
├── src/
│   ├── components/      # Astro 组件
│   │   ├── BaseHead.astro
│   │   ├── Footer.astro
│   │   ├── FormattedDate.astro
│   │   ├── Header.astro
│   │   └── HeaderLink.astro
│   ├── content/
│   │   └── blog/       # 博客文章 (Markdown/MDX)
│   ├── layouts/        # 页面布局
│   │   └── BlogPost.astro
│   ├── pages/          # 路由页面
│   │   ├── index.astro
│   │   ├── about.astro
│   │   ├── rss.xml.js
│   │   └── blog/
│   ├── styles/          # 全局样式
│   │   └── global.css
│   └── consts.ts       # 常量定义
├── docs/                # 项目文档 (本文档所在)
├── astro.config.mjs    # Astro 配置
├── wrangler.json       # Cloudflare Wrangler 配置
└── package.json        # 项目依赖
```

### 3.3 内容管理

- 博客文章存放在 `src/content/blog/` 目录
- 使用 Frontmatter 定义元数据 (title, description, pubDate 等)
- 支持纯 Markdown 和 MDX 格式

### 3.4 环境配置

- API 密钥等敏感信息存放在 `.env` 文件
- `.env` 文件已加入 `.gitignore`，不会提交到远程
